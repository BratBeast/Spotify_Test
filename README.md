## 📂 Лабораторна робота №2: Автоматизоване UI-тестування
**Об'єкт тестування:** Веб-застосунок Spotify Web Player.
**Інструментарій:** Python 3.12, Pytest, Selenium WebDriver.

### Реалізовані сценарії:
1. **Search Functional:** Перевірка коректності роботи пошуку. Автоматичне введення запиту "Imagine Dragons" та валідація результатів у DOM.
2. **Page Load Check:** Перевірка стабільності завантаження головної сторінки та наявності ключових елементів керування.

### Результати:
- Використано механізм **Explicit Waits** для стабілізації тестів в умовах динамічного рендерингу.
- Тести успішно пройдено (**2 Passed**).
- Звіт у форматі HTML: `report_ui.html`

## 🌐 Лабораторна робота №3: Автоматизоване тестування API
**Об'єкт тестування:** Spotify Web API (Endpoints: `/search`, `/tracks`).
**Інструментарій:** Python, Requests, Pytest.

### Реалізовані сценарії:
1. **Unauthorized Search:** Валідація системи безпеки. Перевірка повернення статусу `401 Unauthorized` при відсутності токена.
2. **Invalid Track ID:** Тестування обробки некоректних ідентифікаторів ресурсів.
3. **Security Priority:** Дослідження логіки сервера, де статус `401` має пріоритет над статусом `404 Not Found` для неавторизованих запитів.

### Результати:
- Тести підтвердили коректну обробку HTTP-статусів та цілісність структури JSON-відповідей.
- Тести успішно пройдено (**3 Passed**).
- Звіт у форматі HTML: `report_api.html`
